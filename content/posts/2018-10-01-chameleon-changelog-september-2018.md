<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Great news in Chameleon-land! </span></b></p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"> </p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">We spent the beautiful month of September dreaming up new features and modifications that you see listed below. </span></b></p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"> </p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 700; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Jupyter Notebook Support.</span><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> Some of you may have heard of </span><a href="http://jupyter.org/" style="text-decoration: none;"><span style="">Jupyter notebooks</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> -- with Jupyter, you can provision and configure your testbed resources, run your experiment, download and process its output repeatedly, and generate paper-ready visualizations -- all within one programmable interface. To make it easier for you to use those notebooks, we integrated Jupyter with Chameleon: you go to the </span><a href="https://jupyter.chameleoncloud.org/" style="text-decoration: none;"><span style="">Chameleon JupyterHub server</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">, log in with your Chameleon credentials, and a Jupyter Notebook server is created for you. Chameleon Jupyter notebook servers come pre-installed with useful libraries -- such as </span><a href="https://github.com/chameleoncloud/python-chi" style="text-decoration: none;"><span style="">python-chi</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> to access Chameleon services --  that you can leverage to set up your experimental resources on Chameleon and provide an interface to all common Chameleon actions such as allocating or deploying an instance. To make it easier to get started, we provide a sample Jupyter notebook that you can copy and modify to suit your experiments. Take a look at </span><a href="https://chameleoncloud.readthedocs.io/en/latest/technical/jupyter.html" style="text-decoration: none;"><span style="">our documentation</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> and give it a whirl -- and stay tuned for improvements to the service coming up! In the meantime though, please let us know what you think and how we can improve this feature to make it more useful to you.</span></b></p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"> </p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 700; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">New Software Defined Networking Capabilities.</span><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">  We have refined the user-facing abstraction of the BYOC OpenFlow capabilities. Early BYOC adopters at University of Chicago were presented with two logical OpenFlow switches; nodes could be connected to either switch and it was hard to know which port was connected to which node making networking experiments hard to control. We now reconfigured this so that all user OpenFlow controllers will see only one logical switch (with double capacity), all nodes will be connected to this switch, and the ports in that switch will be consistently named so that the the port to node mapping is easy to control. To find out more about these improvements, read </span><a href="https://chameleoncloud.readthedocs.io/en/latest/technical/networks/networks_sdn.html#" style="text-decoration: none;"><span style="">our BYOC documentation</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">.</span></b></p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"> </p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 700; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Usability features.</span><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> Some of you have signalled that you would prefer to enter lease start and its duration rather than start and end during lease creation -- we adapted the interfaces to support this alternative. Also, to make it easier for you to keep track of your reservations, you will now receive an email reminder about advanced reservations due to start the next day and a notification if you have a new active reservation that has not been used for more than 24 hours. Finally, we added a self-update feature to cc-snapshot that will help you leverage all the new features more easily: if cc-snapshot on your appliance is not up to date, it updates itself before executing.</span></b></p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"> </p>

<p dir="ltr" style="line-height: 1.38; margin-top: 0pt; margin-bottom: 0pt;"><b id="docs-internal-guid-dde62f2c-7fff-9659-29e0-d03015f0a7f8" style=""><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Finally, if any of you happen to be at </span><a href="https://www.ieeelcn.org/index.html" style="text-decoration: none;"><span style="">LCN in Chicago</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> this week, one of our users, Divyashri Bhat from the University of Massachusetts who contributed a </span><a href="https://www.chameleoncloud.org/blog/2018/09/20/application-based-qos-support-p4-and-openflow/" style="text-decoration: none;"><span style="">fantastic methodology blog</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> a couple of weeks ago, will have a demo of her work leveraging some of the latest Chameleon features: the </span><a href="https://www.chameleoncloud.org/blog/2018/08/17/software-defined-networking-openflow-chameleon/" style="text-decoration: none;"><span style="">BYOC controller</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> and the Jupyter notebook integration features described above. Please, join her at the </span><a href="https://www.ieeelcn.org/Program_demos.html" style="text-decoration: none;"><span style="">demonstration session</span></a><span style="font-size: 11pt; font-family: Arial; color: rgb(0, 0, 0); background-color: transparent; font-weight: 400; font-style: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;"> on Tuesday,  October 2, 2018 from 15:30 until 17:30 in Salon EFG. </span></b></p>
